# 商业级起名系统数据获取与处理方案

## 一、目标

* 构建 **10万级中文起名词库**，包含：汉字、拼音、字义、风格标签、行业词、负面过滤词。
* 数据真实可靠，可直接用于名字生成、评分和AI增强。
* 支撑多类型起名：个人、公司、品牌、产品等。
* 数据可扩展、可维护。

---

## 二、数据类型与来源

| 数据类型    | 主要用途          | 数据来源                                | 获取方式                        |
| ------- | ------------- | ----------------------------------- | --------------------------- |
| 汉字集     | 基础字库、合法字集合    | Unicode CJK、常用汉字表                   | 官方 Unicode 表 + 开源字库         |
| 拼音      | 名字音韵分析、发音控制   | CEDICT、python-pinyin、pypinyin库      | GitHub/CEDICT 数据库、Python库解析 |
| 中文字义    | 字义分析、评分系统     | CEDICT、新华字典、Open Chinese Dictionary | CEDICT 公开文本、XML/JSON解析      |
| 人名统计    | 名字评分、性别偏向、常用字 | Chinese Names Corpus、各类姓名公开数据       | GitHub 下载，或从公开数据库抓取         |
| 行业/品牌词  | 公司/品牌起名候选     | 规则生成、公开行业词表                         | 自建规则 + 网页采集 + 行业词库          |
| 风格标签    | 名字风格匹配、筛选     | AI/手工分类                             | AI分类/人工标注                   |
| 负面词/禁用词 | 避免不吉利字、歧义     | 网传不吉利字表、成语库                         | 数据抓取 + 人工审核                 |

---

## 三、获取数据的具体方法

### 1️⃣ 汉字库

* **来源**：

  * Unicode CJK：20902基本汉字 + 扩展A/B
  * 常用汉字表（GB2312、GBK、HSK常用字）
* **获取方法**：

  * 下载 Unicode CJK JSON 或 TXT 文件
  * 保留常用汉字 + 扩展常用字
* **注意**：

  * 去除生僻或不常用字（笔画过多、难写）
  * 标记字源：core / extended

---

### 2️⃣ 拼音

* **来源**：

  * CEDICT、pypinyin、mozillazg/pinyin
* **获取方法**：

  * 对每个汉字映射拼音数组（支持多音字）
  * 如果多音字，保留所有拼音
* **注意**：

  * 多音字处理：根据姓名频率或行业规则优先选择常用读音

---

### 3️⃣ 字义

* **来源**：

  * CEDICT（中英双语解释）
  * Open Chinese Dictionary / Xinhua Chinese Dictionary
* **获取方法**：

  * 解析字典 XML/JSON/TXT 文件
  * 映射到每个字
* **清洗**：

  * 去掉不常用、歧义字义
  * 对多义字，根据名字使用场景选择吉利义
  * 保留中文简明释义，增加英文释义（AI可用）

---

### 4️⃣ 人名统计数据

* **来源**：

  * Chinese Names Corpus
  * 国内姓名公开统计（出生人口数据）
* **用途**：

  * 字性别偏向（M/F/N）
  * 常用度
  * 名字组合优先级
* **清洗**：

  * 去掉非常罕见姓氏/字
  * 统计每个字在名字中出现频率

---

### 5️⃣ 行业词和品牌词

* **来源**：

  * 行业词表（科技、AI、网络、教育、医药）
  * 公司名公开数据（可采集）
* **生成方法**：

  * 收集行业核心词 → 拆分字 → 标注风格
  * AI生成或规则生成组合词
* **清洗**：

  * 去掉重复、歧义、负面含义词
  * 与负面词库交叉校验

---

### 6️⃣ 风格标签

* **来源**：

  * 人工分类 + AI分类
* **方法**：

  * 根据字义、行业词、音律打标签
  * 标签示例：modern / luxury / tech / chinese / ai / literary
* **清洗**：

  * 保证每个字/词标签唯一且准确
  * 风格覆盖至少每个词库类别

---

### 7️⃣ 负面词/禁用词

* **来源**：

  * 网传禁用字列表
  * 成语库、歧义字库
* **清洗**：

  * 对照词库，去除不吉利字
  * 标记等级：轻微不适 / 严重不适 / 禁用

---

## 四、数据清洗与处理流程

1. **统一编码**：全部数据转换为 UTF-8
2. **去重**：同一汉字或词只保留一条，合并来源
3. **冲突处理**：

   * 多音字 → 保存所有拼音，增加权重选择规则
   * 多义字 → 根据场景选择吉利义
4. **频率归一化**：

   * 字在姓名中的使用频率，标准化为 0~1
5. **风格标签赋值**：

   * 基于行业、字义、音律自动赋值
6. **负面词过滤**：

   * 与负面列表交叉，标记或删除
7. **行业词/组合词生成**：

   * 按行业/风格规则生成组合词
8. **结构化输出**：

   * JSON / SQLite / CSV → 支持生成引擎直接读取

## 五、数据结构化建议

### 字表

| 字段        | 类型     | 说明       |
| --------- | ------ | -------- |
| char      | string | 汉字       |
| pinyin    | array  | 拼音列表     |
| meaning   | string | 中文义      |
| frequency | float  | 常用度 0~1  |
| gender    | string | M/F/N    |
| vibe      | array  | 风格标签     |
| category  | array  | 核心/扩展/行业 |
| nameScore | float  | 名字适配度评分  |
| source    | array  | 数据来源     |

### 组合词表

| 字段       | 类型     | 说明                 |
| -------- | ------ | ------------------ |
| text     | string | 词/名字               |
| type     | string | prefix/suffix/core |
| industry | array  | 所属行业               |
| vibe     | array  | 风格标签               |
| weight   | float  | 组合权重               |

## 六、数据更新与维护

1. **定期更新**：

   * 新增常用字、流行词、行业词
   * 增加AI自动打标签和评分
2. **异常检查**：

   * 冲突字、多音字异常
   * 负面字误入
3. **版本控制**：

   * 所有数据放在Git或数据库版本中
   * 更新可回滚

---

## 七、数据安全与版权注意

* 使用公开开源数据（CEDICT、Unicode、公开姓名语料）
* 商业使用需核查许可（尤其是姓名数据库）
* AI增强生成内容属于系统产出，可作为增值服务

---

## 八、总结

1. 数据是起名系统的核心资产，必须多源融合：

   * 汉字库 + 拼音 + 字义 + 人名统计 + 行业词 + 风格标签 + 负面词
2. 清洗流程严格：

   * 去重 → 冲突处理 → 频率归一 → 风格标签 → 负面过滤
3. 输出结构化：

   * JSON/SQLite/CSV，可直接驱动生成引擎和评分系统
4. 可扩展性：

   * AI标签、行业词、组合词、国际化
